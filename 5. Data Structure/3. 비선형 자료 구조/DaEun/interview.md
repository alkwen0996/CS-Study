## 📝 Interview

#### 그래프를 구현하는 두 가지 방법에 대해 설명해주세요.
```
인접 행렬과 인접 리스트를 통해 그래프를 구현할 수 있습니다.

인접 행렬은 정밥 행렬을 사용해 그래프의 정점 간 연결 여부를 저장하여 그래프를 표현합니다.
정방 행렬의 i번째 행과 j번째 열에 위치한 요소는 노드 i와 j 사이의 간선 여부를 나타냅니다.
연결 여부에 따라 0, 1를 저장하거나 가중치 그래프의 경우 가중치 값을 저장합니다.

인접 리스트는 각 정점마다 연결된 다른 정점들의 목록을 리스트 형태로 저장하여 그래프를 표현합니다.
인접 행렬은 간선의 수과 관계 없이 N*N 크기의 메모리가 필요한데,
인접 리스트는 연결된 정점들에 대한 메모리만 필요하기 때문에 메모리 사용 측면에서 효율적입니다.

이런 특성상 인접 행렬은 간선의 수가 많은 경우 유용하고,
인접 리스트는 간선이 적은 경우에 적합합니다.
```

#### 그래프와 트리의 차이점에 대해 설명하세요.
```
그래프와 트리 모두 정점과 간선으로 이루어진 비선형 자료 구조입니다.
트리는 그래프의 특별한 형태로 사이클이 없으며 계층적인 구조, 비순환적인 구조를 가진 그래프를 트리라고 부릅니다.

그래프와 달리 트리에선 임의의 두 노드 사이엔 반드시 하나의 단순 경로가 존재합니다.
```
<br>

#### 이진 트리의 종류 중 완전 이진 트리, 포화 이진 트리, 정이진 트리에 대해 설명하세요.
```
이진 트리는 자식의 노드 수가 최대 두 개인 트리를 의미하는데,
형태에 따라 여러 가지 종류로 구분할 수 있습니다.

완전 이진 트리는 왼쪽 부터 채워져 있는 이진트리를 말하며
마지막 레벨을 제외하고 모든 레벨의 노드가 완전히 채워져 있습니다.
마지막 레벨에서도 왼쪽부터 채워져 있으며 힙 트리에 이용되는 구조입니다.

포화 이진 트리는 모든 레벨의 모든 노드가 채워져 있는 이진 트리입니다.
이진 트리는 트리의 높이(n)에 따라 최대 2^n - 1개의 노드 수를 가질 수 있는데
포화 이진 트리의 노드 수가 이에 해당합니다.

정이진 트리는 각 노드에 대해 자식 노드가 없거나 두 개인 이진 트리를 말합니다. 
```
<br>

#### 이진 트리와 이진 탐색 트리의 차이점에 대해 설명해주세요.
```
이진 트리는 단순히 자식의 노드 수가 최대 두 개인 트리 구조를 말합니다.

이진 탐색 트리는 이진 트리의 한 종류로 이진 트리에 추가적인 조건을 가집니다.
각 노드의 왼쪽 서브 트리는 해당 노드보다 작은 값들이 저장되어 있고,
오른쪽 서브 트리에는 해당 노드보다 큰 값들이 저장되어 있습니다.

이런 특징을 기반으로 이진 탐색을 통해 효율적인 검색을 가능하게 합니다.
특정 값을 검색하는데 평균 O(logN)의 시간 복잡도를 가집니다.
하지만 선형적인 구조를 가진 이진 탐색 트리에선 최악의 시간 복잡도를 가진다는 단점이 있습니다.
```

<br>

#### 힙 자료구조에 대해 설명해주세요.
```
노드가 왼쪽부터 채워져 있는 완전 이진 트리를 기반으로 한 자료구조로,
각 노드가 자식 노드보다 크거나 작은 값이 들어가야 한다는 조건으로 정렬이 되어있는 구조를 말합니다.

부모노드에 큰 값이 들어가야 하는 구조를 최대힙이라고 하며,
작은 값이 들어가야 하는 구조를 최소힙이라고 합니다.

루트 노드엔 항상 전체 트리에 대한 최솟값이나 최댓값이 들어가 있기 때문에
빠른 최소,최대값 검색이 가능합니다. 우선 순위 큐 구현이나 힙 정렬에 활용됩니다.
```

#### 힙에서 데이터 삽입, 삭제 과정에 대해 설명하세요.
```
데이터 삽입 시 완전 이진 트리의 특징에 따라 마지막 레벨의 가장 왼쪽에 노드를 추가합니다.
그 후 부모노드와 값을 비교하며 최소힙일 경우 부모노드보다 작으면, 최대힙일 경우 부모노드보다 클때
부모와 자리를 바꿉니다. 이 과정을 트리가 제대로 정렬될 때까지 반복합니다.

힙에서 데이터 삭제는 루트 노드만 제거할 수 있습니다.
루트 노드를 삭제하고 마지막 노드를 루트 노드로 올립니다.
그 이후 자식 노드와 비교, 스왑하며 이동한 노드가 제자리를 찾아갈 수 있도록 합니다.
이 때, 다른 노드가 아닌 마지막 노드를 루트로 이동시키는 이유는
힙이 완전 이진 트리 구조를 가지기 때문에 중간 노드를 올리면 다른 노드들이 빈 칸을 채우기 위해
이동하기 때문입니다. 즉, 다른 노드의 이동을 최소화하기 위해 마지막 노드를 올립니다.
```
